# Role Onboarding Checklist

This checklist helps new team members understand their role responsibilities and key collaboration patterns at OctoAcme. Refer to [Roles & Personas](octoacme-roles-and-personas.md) for detailed role descriptions.

---

## General Onboarding (All Roles)

- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles & Personas](octoacme-roles-and-personas.md) documentation
- [ ] Understand communication cadences (standups, syncs, demos)
- [ ] Access project boards and collaboration tools
- [ ] Join relevant Slack/Teams channels
- [ ] Review current project charter and roadmap

---

## Developer Onboarding

- [ ] Set up local development environment
- [ ] Review code style guides and PR conventions
- [ ] Understand branching strategy and git workflow
- [ ] Configure CI/CD pipeline access
- [ ] Complete first code review (as reviewer)
- [ ] Submit first PR (small/documentation change)
- [ ] Meet with QA to understand testing expectations
- [ ] Review technical design documentation

**Key Collaborators**: QA Engineers, DevOps Engineers, UX Designers, Product Managers

---

## Product Manager Onboarding

- [ ] Review product vision and strategy documents
- [ ] Understand current roadmap and backlog prioritization
- [ ] Meet with Business Analysts to review requirements process
- [ ] Review customer feedback and feature request backlog
- [ ] Understand success metrics and KPIs
- [ ] Shadow customer calls or support interactions
- [ ] Meet with UX to understand design process
- [ ] Review release planning process

**Key Collaborators**: Business Analysts, UX Designers, Customer Success, Developers

---

## Project Manager Onboarding

- [ ] Review active projects and their status
- [ ] Understand risk register and escalation paths
- [ ] Access project tracking tools (GitHub Projects, etc.)
- [ ] Review meeting facilitation guidelines
- [ ] Shadow project planning and retrospective sessions
- [ ] Understand stakeholder communication expectations
- [ ] Review resource allocation and capacity planning
- [ ] Meet with all role leads to understand team structure

**Key Collaborators**: All roles (coordination and facilitation)

---

## QA/Testing Engineer Onboarding

- [ ] Set up test environments and automation frameworks
- [ ] Review test plans and coverage requirements
- [ ] Understand Definition of Done and acceptance criteria format
- [ ] Access bug tracking and test management tools
- [ ] Review CI/CD test integration
- [ ] Shadow exploratory testing session
- [ ] Meet with Developers to understand code review process
- [ ] Review quality metrics and reporting expectations

**Key Collaborators**: Developers, DevOps Engineers, Product Managers

---

## UX Designer Onboarding

- [ ] Review design system and component library
- [ ] Access design tools (Figma, Sketch, etc.)
- [ ] Understand user research processes and schedules
- [ ] Review accessibility standards and testing process
- [ ] Meet with Product Managers to understand product strategy
- [ ] Shadow usability testing session
- [ ] Review design handoff and specification process
- [ ] Understand feedback collection from Customer Success

**Key Collaborators**: Product Managers, Developers, Business Analysts, Customer Success

---

## DevOps Engineer Onboarding

- [ ] Access cloud infrastructure and deployment systems
- [ ] Review CI/CD pipeline architecture
- [ ] Understand monitoring and alerting setup
- [ ] Review deployment procedures and rollback plans
- [ ] Access runbooks and incident response documentation
- [ ] Meet with Developers to understand build requirements
- [ ] Review security scanning and compliance requirements
- [ ] Understand capacity planning and scaling policies

**Key Collaborators**: Developers, QA Engineers, Project Managers

---

## Business Analyst Onboarding

- [ ] Review requirements documentation standards
- [ ] Understand stakeholder mapping and engagement process
- [ ] Access business case templates and ROI analysis tools
- [ ] Shadow requirements gathering workshop
- [ ] Meet with Product Managers to understand strategy
- [ ] Review user story and acceptance criteria formats
- [ ] Understand process analysis and modeling tools
- [ ] Review requirements traceability approach

**Key Collaborators**: Product Managers, Project Managers, Developers, UX Designers

---

## Customer Success / Support Engineer Onboarding

- [ ] Access support ticketing system and knowledge base
- [ ] Review customer onboarding and training materials
- [ ] Shadow customer interactions and support calls
- [ ] Understand escalation process for bugs and feature requests
- [ ] Meet with Product Managers to understand product roadmap
- [ ] Review customer health metrics and reporting
- [ ] Understand SLA requirements and response time expectations
- [ ] Access customer communication templates

**Key Collaborators**: Product Managers, UX Designers, QA Engineers, Developers

---

## Cross-Role Collaboration

All team members should:

- [ ] Review [Role Interaction Matrix](octoacme-roles-and-personas.md#role-interaction-matrix)
- [ ] Understand when to escalate vs. resolve issues directly
- [ ] Know how to request help from other roles
- [ ] Participate in cross-functional retrospectives
- [ ] Contribute to process improvement discussions
